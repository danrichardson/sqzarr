## Description
The history tab accumulates completed jobs indefinitely. Users should be able to clear old "Done" entries from the history view to keep it manageable, but clearing history must NOT cause those files to be picked up again by the scanner and re-processed.

This is directly related to TKT-004 (files reappearing after being handled). The underlying requirement is that "processed" state must be tracked separately from "visible in history" — history is a UI concern, processing state is a data concern.
