## Description
The Settings panel shows the active hardware encoder (e.g., "Intel VAAPI (hevc_vaapi)") but doesn't let the user choose between options. Two things are needed:

1. **Multiple hardware encoders**: If the system has more than one hardware encoder available (e.g., VAAPI + NVENC, or multiple GPU devices), the user should be able to select which one to use from a dropdown in the settings panel.
2. **Software encoding fallback**: The user should have the option to enable software encoding (e.g., libx265) even though it's slow. This is useful as a fallback or for environments without hardware acceleration.

Investigate whether the backend already detects multiple encoders — it may just be a UI issue of not exposing the choice. The current user only has one HW encoder so this may already be partially handled on the backend.
