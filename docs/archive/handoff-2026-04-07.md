# SQZARR Handoff — 2026-04-07

Session focused on UX/safety improvements to Settings, Directories, and the Queue
file picker. All changes built and deployed to the LXC at 192.168.29.211 via
`tar | ssh` → server-side `go build` → systemctl restart.

## What shipped this session

### 1. Root Directories feature (Settings)
The "filesystem safe zone" is now its own first-class concept, separate from
scan directories.

- **Backend** (`internal/config/config.go`): added `ScannerConfig.RootDirs []string`.
- **Backend** (`internal/api/handlers.go`):
  - `validateDirPath()` — exists + isDir + writable (sentinel file test) +
    within-a-root. Wired into `handleCreateDirectory` and `handleUpdateDirectory`
    so arbitrary paths can no longer be saved.
  - `handleBrowseFS` uses `Scanner.RootDirs` as the picker ceiling. Defaults to
    first root, suppresses `parent` at root boundaries.
  - `handleGetConfig` exposes `root_dirs`; `handleUpdateConfig` accepts/validates
    them and writes them as a TOML inline array.
- **Frontend** (`Settings.tsx`):
  - New "Root Directories" card with add/remove UI.
  - Saved as part of the main "Save Settings" form.
  - Browse button next to the add input opens an **unrestricted** picker
    (`RootDirBrowser` component) that bypasses the root ceiling — this is the one
    place you need free filesystem navigation since you're defining the roots.

### 2. Unrestricted browse mode
- `GET /fs?unrestricted=1` bypasses both the ceiling check and the parent
  suppression. Used only by the Settings root-dir picker.
- `api.browseFS(path, files, unrestricted)` third param.

### 3. Settings: dirty tracking + nav guard
- `isDirty` (`useMemo`) compares every form field to the loaded `cfg`. Root dirs
  use sorted-JSON comparison.
- "Save Settings" button is disabled until something actually changes.
- **In-app navigation**: `useBlocker(isDirty)` from React Router shows a custom
  "Unsaved changes — Stay / Leave anyway" modal when navigating away.
- **Browser close/back**: `beforeunload` listener triggers the native prompt.

### 4. Router migration (data router)
- `useBlocker` only works with data routers, **not** legacy `<BrowserRouter>`.
  Caused a blank Settings page until fixed.
- `App.tsx` migrated from `<BrowserRouter><Routes>` to
  `createBrowserRouter([...])` + `<RouterProvider>`. Layout's `<Outlet>` was
  already in place so no other changes needed.

### 5. Directory delete bug
- **Symptom**: clicking delete in Directories did nothing.
- **Root cause**: `_foreign_keys=on` is set on the SQLite connection.
  `jobs.directory_id` and `scan_runs.directory_id` both have FK refs to
  `directories(id)` with no cascade rule. Any directory with jobs/runs would fail
  the delete with a constraint violation. The frontend silently swallowed the
  500 in `catch {}`.
- **Fix** (`internal/db/directories.go`): `DeleteDirectory` now runs in a
  transaction — NULL out `directory_id` on referencing jobs and scan_runs first
  (preserves history per the confirm dialog's promise), then delete the row.
- Also surfaced the error to the user via `setError` instead of swallowing.

### 6. File picker multi-select (Queue)
- `FilePicker` rewritten with `selected: Set<string>` state.
- Checkbox indicator on each file row; click toggles selection (no longer
  closes the modal). Selected rows highlight amber.
- "Select all / Deselect all" link in the path bar.
- Footer with "N of M selected" + "Queue N files" button.
- Selection clears when navigating to a different directory.
- Parent handler queues each selected path sequentially via `api.createJob`,
  reports failures aggregated, then refreshes.

## Deployment

Source synced to `/tmp/sqzarr_src` on 192.168.29.211 via `tar | ssh`. `go.mod`
and `go.sum` were regenerated on the server (Go isn't installed on the Windows
dev box) and synced back. Binary at `/usr/local/bin/sqzarr`, service is
`systemctl sqzarr` and is currently `active (running)`.

## Known follow-ups / not done

- `sqzarr.toml.example` still doesn't document the `scanner.root_dirs` field.
- The directory picker on the **Edit / Add Directory** form (Directories.tsx)
  still uses the restricted `browseFS` — correct, but worth a note that you must
  configure roots in Settings *first* or that picker rejects the page.
- Frontend still has `catch {}` swallowing in a few places worth auditing
  (`load()` in Directories, etc.).
- Tar uploads from Windows produce ownership warnings that look alarming but are
  cosmetic — files extract fine. Could switch to `--owner=root --group=root` to
  silence them.
